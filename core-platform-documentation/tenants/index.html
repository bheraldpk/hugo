Multi-Tenant Setup

Determining the Tenant

The platform will use the following order to determine the tenant for the user:

  1. The tenants claim in the id token
    • If there are multiple, the last one in the list is used
    • To add the tenant claim, find the application in Okta and click modify next to the user on the assignments tab
  2. The default tenant (if configured)

Warning The x-tenant-id header is now removed from the request headers in the kong cluster plugin

Environment

The environment is determined based on a lua template in the kong cluster plugin.
The x-environment header will be added based on the URL.

Note This value can’t be overwritten in the request headers, the value is removed from the request headers if present and always computed from the url.

Adding a Tenant

  1. Register a tenant in the system
    • Configure a http port forward (5051) from K9s or from the command line
    • Open the swagger api (localhost:<port>/q/swagger-ui) and use the registerTenant API
  2. Add the settings in a folder structure of tenant-manager/applications/{app}/{environment}/settings.json for the various applications (services).
    • See/deployment/gcp/dev for an example
  3. Configure a GRPC port forward (5050) and load the application settings
 igor tenant sync-settings --dir=$(PWD)/tenant-manager/applications --port=<local port> -n=<tenant id>
  1. Add the database secret to vault:
# Connect to the pod
kubectl exec -it --namespace=<namespace> vault-0 -- /bin/sh

# Add the secret
vault login root
vault kv put secret/<tenant>/<environment>/database postgres-user=admin postgres-password=<password>

  1. Add the new tenant in Okta for the appropriate users.

  2. Create the schema in postgres

# Start a debug pod
kubectl run -i --tty --rm debug --image=postgres:alpine --restart=Never --namespace=dev -- sh

# Login to psql
PGPASSWORD=admin psql -h <Database IP> -U admin postgres

# Connect to database
\c infinity

# Create schema
CREATE SCHEMA <schema name from configs>;

Note: any tables created in the admin-ui won’t display in psql until the first connection is made. Adding data to the table will create it.

When using a schema other than the default, you may have to set the search path to see the new tables

set search_path to <schema name from configs>;
